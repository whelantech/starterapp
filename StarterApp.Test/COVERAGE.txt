Coverage and quality metrics (SET09102 / SonarQube Cloud)

- CI: GitHub Actions runs `dotnet-coverage collect "dotnet test StarterApp.sln -c Release -f net10.0 --no-build" -f xml -o "coverage.xml"` and uploads to Sonar (`sonar.cs.vscoveragexml.reportsPaths=coverage.xml`).
- Local (same as CI): `dotnet build StarterApp.sln -c Release -f net10.0` then `dotnet-coverage collect "dotnet test StarterApp.sln -c Release -f net10.0 --no-build" -f xml -o coverage.xml`.
- Assemblies exercised by tests include StarterApp (ViewModels, Services, Converters) and StarterApp.Database (repositories, workflow, API fakes) when referenced from StarterApp.Test.
- Optional Sonar exclusions for boilerplate (tune in SonarCloud UI or begin step): `**/Platforms/**`, `**/Resources/**`, `**/obj/**`, `**/Migrations/**`.
- Rubric alignment: combine fast unit tests (AAA, mocked IApiService via ApiTestService) with EF InMemory fixture tests and optional PostgreSQL integration when `ConnectionStrings__TestConnection` / `TEST_DATABASE_CONNECTION` matches your database.
